# MaritimeBench 航运知识问答系统 - 使用手册

**版本**: v0319 (2026-03-18)
**项目**: MaritimeBench 航运知识问答系统
**技术**: RAG（检索增强生成）+ Qwen3-8B

---

## 一、系统简介

本系统是基于 RAG 技术的航运知识问答系统，通过检索航运专业文档库并结合 Qwen3-8B 大模型，实现对航运领域单选题的智能回答。系统已构建包含 5 个航运专业 PDF 文档（97.5MB）的向量知识库，可直接使用。

**核心优势**:
- 知识增强：检索航运专业知识提升准确率
- 即开即用：向量库已构建完成
- 多种模式：支持交互式问答和批量推理
- 格式规范：自动输出 `[A]`/`[B]`/`[C]`/`[D]` 格式答案

## 二、快速开始

### 2.1 环境准备

**系统要求**:
- Python 3.8+
- 内存 2GB+
- 磁盘空间 500MB+

**安装依赖**:
```bash
cd RAG
pip install -r requirements.txt
```

主要依赖包：
- `openai>=1.0.0` - API 调用
- `langchain>=0.1.0` - LangChain 核心库
- `langchain-community>=0.0.20` - 文档处理和向量存储
- `faiss-cpu>=1.7.4` - 向量检索
- `sentence-transformers>=2.2.0` - 文本嵌入
- `pypdf>=3.0.0` - PDF 解析
- `modelscope>=1.11.0` - 模型下载

### 2.2 配置 API Key

**方式一：环境变量（推荐）**
```bash
export DASHSCOPE_API_KEY="your-api-key-here"
```

**方式二：修改代码**
在 [answer.py](answer.py) 第 22 行修改：
```python
API_KEY = "your-api-key-here"
```

### 2.3 离线模式配置

如遇网络连接问题（HuggingFace 模型下载超时），使用离线模式：

**Windows 命令行**:
```bash
set TRANSFORMERS_OFFLINE=1
set HF_DATASETS_OFFLINE=1
set PYTHONIOENCODING=utf-8
python generate_jsonl_rag.py
```

**Linux/Mac**:
```bash
export TRANSFORMERS_OFFLINE=1
export HF_DATASETS_OFFLINE=1
export PYTHONIOENCODING=utf-8
python generate_jsonl_rag.py
```

**说明**:

- `TRANSFORMERS_OFFLINE=1` - 禁用模型在线检查
- `HF_DATASETS_OFFLINE=1` - 禁用数据集在线检查
- `PYTHONIOENCODING=utf-8` - 解决 Windows 编码问题

## 三、使用方式

### 3.1 交互式问答（推荐用于测试）

**启动系统**:
```bash
python answer.py
```

**使用流程**:
1. 输入题目文本
2. 依次输入选项 A、B、C、D（可直接回车跳过）
3. 系统自动检索知识库并给出答案
4. 显示答案和参考资料（前 300 字符）
5. 输入 `quit`、`exit` 或 `q` 退出

**示例**:
```
📝 请输入题目（或输入 quit 退出）: 船舶电力系统供电网络中，放射形网络的特点是______。

请输入选项（直接回车跳过）:
  A. ②③
  B. ①③
  C. ②④
  D. ①④

🤔 正在思考...

============================================================
✅ 答案: [B]

📚 参考资料:
------------------------------------------------------------
放射形网络是指从配电板向各用电设备呈发散形传输的供电方式...
============================================================
```

### 3.2 批量推理模式（用于正式评测）

**启动批量模式**:
```bash
python answer.py --batch
```

**功能说明**:
- 批量处理预定义的题目列表
- 自动生成 `predictions.jsonl` 提交文件
- 显示处理进度
- 适合处理 1888 道题目

**修改题目列表**:
编辑 [answer.py](answer.py) 第 96-110 行的 `sample_questions` 列表：
```python
sample_questions = [
    {
        'question': '题目文本',
        'options': {'A': '选项A', 'B': '选项B', 'C': '选项C', 'D': '选项D'}
    },
    # 添加更多题目...
]
```

**输出格式**:
```json
{"index": 0, "input": "题目文本", "prediction": "[B]", "context_used": "参考资料..."}
{"index": 1, "input": "题目文本", "prediction": "[A]", "context_used": "参考资料..."}
```

### 3.3 简化版（无 RAG）

**用途**: 对比测试、快速验证 API 连接

**启动**:
```bash
python answer_simple.py
```

**特点**:
- 不使用知识库检索
- 直接调用 Qwen3-8B 模型
- 预期准确率 40-50%
- 响应速度更快

### 3.4 Web 界面

**启动 Web 应用**:
```bash
python app.py
```

**访问**: http://127.0.0.1:5000

**注意**: 当前 Web 应用尚未集成 RAG 功能，仅提供基础对话能力。

## 四、知识库管理

### 4.1 查看知识库状态

**知识库位置**: `RAG/vectorstore/`
- `index.faiss` - FAISS 向量索引（6.9MB）
- `index.pkl` - 向量元数据（3.7MB）

**文档库位置**: `knowledge_docs/`
- 船舶动力装置书2012新版.pdf（19.5MB）
- 船舶电气与自动化.pdf（14.7MB）
- 船舶管理2012版.pdf（21.4MB）
- 船舶维修技术实用手册.pdf（32.0MB）
- 船舶辅机.pdf（9.9MB）
- 避碰选择题题库.pdf（参考资料）

**支持的文档格式**:
- PDF - 通过 PyPDF 解析
- TXT - 纯文本文件
- DOCX - Word 文档（新增支持）
- JSON - 结构化数据（问答对、知识库等）

### 4.2 重建知识库（可选）

**场景**: 添加新文档或修改分块参数

**步骤**:
1. 将新的文档放入 `knowledge_docs/` 目录（支持 PDF、TXT、DOCX、JSON）
2. 运行构建脚本：
```bash
cd RAG
python rag_builder.py
```

**构建参数**（在 [rag_builder.py](RAG/rag_builder.py) 中修改）:
- `chunk_size=500` - 文本块大小
- `chunk_overlap=50` - 块重叠大小
- `model_name='BAAI/bge-small-zh-v1.5'` - 嵌入模型

**JSON 格式支持**:
- 列表格式：`[{"question": "...", "answer": "...", "A": "...", ...}, ...]`
- 字典格式：`{"key1": "value1", "key2": "value2", ...}`

**注意**: 首次运行会自动下载 BGE 模型（约 400MB），需要网络连接。

### 4.3 测试知识库检索

运行构建脚本后会自动测试检索功能：
```bash
cd RAG
python rag_builder.py
```

输出示例：
```
🔍 测试检索功能...
查询: 船舶电力系统
结果数量: 2

结果 1:
船舶电力系统是船舶的重要组成部分，主要包括发电机组、配电板...
```

## 六、常见问题

### 6.1 API 调用失败

**错误**: `❌ API 调用失败: Unauthorized`

**解决**:
1. 检查 API Key 是否正确
2. 确认 API Key 有效期和额度
3. 检查网络连接

### 6.2 向量库加载失败

**错误**: `⚠️ 向量库不存在: ./RAG/vectorstore`

**解决**:
1. 确认 `RAG/vectorstore/` 目录存在
2. 检查目录中是否有 `index.faiss` 和 `index.pkl` 文件
3. 如果缺失，运行 `python RAG/rag_builder.py` 重建

**自动降级**: 系统会自动切换到无 RAG 模式继续运行

### 6.3 答案格式不正确

**问题**: 模型输出解释文字而非纯答案

**原因**: Prompt 控制不稳定

**解决**:
- 系统已内置正则表达式提取答案
- 如果提取失败，默认返回 `[A]`
- 可在 [rag_engine.py](RAG/rag_engine.py) 第 122-142 行修改提取逻辑

### 6.4 内存不足

**错误**: `MemoryError` 或系统卡顿

**解决**:
1. 关闭其他占用内存的程序
2. 减少检索数量 k（在 [rag_engine.py](RAG/rag_engine.py) 第 66 行修改）
3. 使用更小的嵌入模型

### 5.5 推理速度慢

**原因**:
- 向量检索时间（< 50ms）
- API 调用时间（2-4 秒）
- 网络延迟

**优化**:
- 使用更快的网络环境
- 批量推理时考虑并发调用（需注意 API 限流）
- 减少检索数量 k

## 六、高级配置

### 6.1 调整检索参数

在 [rag_engine.py](RAG/rag_engine.py) 第 66 行修改：
```python
def retrieve_knowledge(self, question, k=3):  # k 为检索数量
```

**建议值**:
- k=1: 速度快，但可能遗漏信息
- k=3: 平衡（默认）
- k=5: 信息全面，但可能引入噪声

### 6.2 修改 Prompt

**系统提示词**: [rag_engine.py](RAG/rag_engine.py) 第 52-57 行
```python
self.system_prompt = """你是航运领域专家。请根据提供的参考资料回答单选题。

要求：
1. 只输出选项字母（A/B/C/D）
2. 必须用方括号包裹，如 [A]
3. 不要输出任何解释或额外内容"""
```

**用户提示词**: [rag_engine.py](RAG/rag_engine.py) 第 84-120 行

### 6.3 更换嵌入模型

在 [rag_builder.py](RAG/rag_builder.py) 和 [rag_engine.py](RAG/rag_engine.py) 中修改：
```python
embeddings = HuggingFaceEmbeddings(
    model_name='BAAI/bge-small-zh-v1.5',  # 更换为其他模型
    model_kwargs={'device': 'cpu'}
)
```

**推荐模型**:
- `BAAI/bge-small-zh-v1.5` - 384维，速度快（默认）
- `BAAI/bge-base-zh-v1.5` - 768维，效果更好但更慢
- `BAAI/bge-large-zh-v1.5` - 1024维，效果最好但最慢
